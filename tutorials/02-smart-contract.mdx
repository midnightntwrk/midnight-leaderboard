
The `compact` script invokes the Compact compiler. The `build` script compiles TypeScript and copies the managed output into `dist/`.

## Write the contract

Create the contract source file. This file contains the entire Compact smart contract.

```bash
touch contract/leaderboard.compact
```

The contract defines a `ScoreEntry` struct stored in a `Map` keyed by an auto-incrementing counter. A `witness` function lets the frontend feed private data into the circuit on demand.

```typescript
pragma language_version >= 0.23;

import CompactStandardLibrary;

struct ScoreEntry {
  score: Uint<64>,
  displayName: Bytes<32>,
  ownerHash: Bytes<32>
}

export ledger scores: Map<Uint<64>, ScoreEntry>;
export ledger nextId: Counter;

witness getCustomName(): Bytes<32>;
```

`scores` is the public ledger readable from the indexer. `nextId` auto-increments to assign unique entry IDs. The `witness` declaration tells the compiler that `getCustomName` will be provided at runtime by the TypeScript host.

### submitScore

```typescript
export circuit submitScore(
  score: Uint<64>,
  useCustomName: Boolean
): [] {
  const userPublicKey = ownPublicKey();
  const ownerHash = persistentHash<Bytes<32>>(userPublicKey.bytes);

  nextId.increment(1);
  const entryId = disclose(nextId.read() as Uint<64>);

  if (disclose(useCustomName)) {
    const customName = getCustomName();
    scores.insert(entryId, ScoreEntry {
      score: disclose(score),
      displayName: disclose(customName),
      ownerHash: disclose(ownerHash)
    });
  } else {
    scores.insert(entryId, ScoreEntry {
      score: disclose(score),
      displayName: disclose(persistentHash<Bytes<32>>(userPublicKey.bytes)),
      ownerHash: disclose(ownerHash)
    });
  }
}
```

`ownPublicKey()` returns the caller's public key, only visible inside the circuit. It never appears on-chain unless explicitly disclosed.

`persistentHash` runs a Poseidon hash. The result is stored as `Bytes<32>` instead of `Field` to avoid overflow issues with the BLS12-381 scalar field.

`disclose()` moves data from private (inside the circuit) to public (on-chain). Every value in a ledger write must be disclosed.

The `if/else` branch controls what gets stored as the display name. When `useCustomName` is true, the contract calls the `getCustomName()` witness to retrieve a name from the TypeScript host. This is used for both custom-named and public-address submissions, with the application layer deciding which value to feed through the witness. When `useCustomName` is false, no witness is invoked and the display name is set to `persistentHash(publicKey)`, producing raw hash bytes that the UI renders as a generated name like "Crimson Tiger."

### verifyOwnership

```typescript
export circuit verifyOwnership(targetEntryId: Uint<64>): [] {
  assert(scores.member(disclose(targetEntryId)), "entry not found");
  const entry = scores.lookup(disclose(targetEntryId));
  const callerHash = persistentHash<Bytes<32>>(ownPublicKey().bytes);
  assert(callerHash == entry.ownerHash, "not the owner");
}
```

The caller proves that `hash(theirPublicKey) == entry.ownerHash` without revealing the public key itself. The proof is generated locally and can be used to claim a prize, verify identity, or present to a badge system. It does not write anything to the ledger.

### getEntryCount

```typescript
export circuit getEntryCount(): Uint<64> {
  return nextId.read() as Uint<64>;
}
```

## Compile the contract

Install the project dependencies and compile the contract. The compiler generates TypeScript bindings, circuit keys, and ZKIR files in `contract/managed/leaderboard/`.

```bash
npm install
npm run compile
```

Expected output:

```
Compiling 3 circuits:
  circuit "getEntryCount" (k=6, rows=26)
  circuit "submitScore" (k=13, rows=4720)
  circuit "verifyOwnership" (k=13, rows=2352)
```
