# Building a Privacy-Preserving Leaderboard DApp

This tutorial walks you through building a full-stack DApp on the Midnight Network. You will write a Compact smart contract, integrate it with TypeScript, build a browser-based frontend with Lace wallet support, and deploy the finished application to production.

By the end, you will have a working arcade-style leaderboard where players submit scores with privacy controls and prove ownership of entries using zero-knowledge proofs.

## What you will learn

**Smart contract (Compact):** defining structured on-chain data with `Map`, `Counter`, and custom structs; controlling what data goes on-chain with `disclose()`; using witnesses to feed private data into ZK circuits on demand; using `assert` to enforce ownership checks; hashing with `persistentHash` for anonymous identity.

**TypeScript integration:** compiling Compact contracts and working with the generated TypeScript bindings; building a platform-agnostic API layer with `midnight-js-contracts`; creating witness providers that bridge private data into the circuit.

**Browser DApp:** connecting to the Lace wallet via the DApp Connector API; initializing providers (proof, indexer, wallet, ZK config); reading on-chain state directly from the indexer without a wallet; submitting transactions through the browser with ZK proof generation.

**Production deployment:** deploying the frontend to Vercel; running a proof server on Railway for CORS-safe proving; configuring environment variables for different networks.

## Prerequisites

**Required:**
- [Node.js v22+](https://nodejs.org/) (use `nvm install 22` if needed)
- [Compact toolchain](https://docs.midnight.network/getting-started/installation#install-compact) (compiler version 0.31.0)
- [Docker](https://docs.docker.com/desktop/) (for the local proof server)
- [Lace wallet](https://chromewebstore.google.com/detail/lace/gafhhkghbfjjkeiendhlofajokpaflmk) browser extension

**Wallet setup:**
1. Install the Lace extension and create a new wallet
2. Set the network to **Preprod**
3. Set the proof server to **http://localhost:6300**
4. Fund your wallet with tNIGHT from the [Preprod faucet](https://faucet.preprod.midnight.network/)
5. Go to Tokens, click **Generate tDUST**, and confirm the transaction

> **Note:** tDUST is required to pay transaction fees on Preprod. It is generated from your tNIGHT balance.

## Project structure

```
midnight-leaderboard/
├── contract/              # Compact smart contract + TypeScript bindings
│   ├── leaderboard.compact
│   ├── src/
│   │   ├── index.ts       # Compiled contract exports + witness wiring
│   │   └── witnesses.ts   # Witness provider
│   └── managed/           # Compiler output
├── api/                   # Shared business logic (platform-agnostic)
│   └── src/
│       ├── index.ts       # LeaderboardAPI class
│       ├── common-types.ts
│       └── utils/index.ts
├── leaderboard-ui/        # React + Vite browser DApp
│   └── src/
│       ├── App.tsx         # Game UI + leaderboard + wallet connection
│       ├── main.tsx        # Entry point with Buffer polyfill
│       ├── contexts/       # Lace wallet provider bridge
│       └── hooks/          # Indexer read hook
├── proof-server/           # Docker config for Railway deployment
│   └── Dockerfile
├── vercel.json             # Frontend deployment config
└── package.json            # Workspace root
```
